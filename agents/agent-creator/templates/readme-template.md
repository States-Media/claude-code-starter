## Usage Instructions

### How to Use This Template

1. **Replace all placeholders** in [brackets]
2. **Maintain markdown formatting** (headers, lists, code blocks)
3. **Ensure minimum length** of 250+ lines
4. **Include specific examples** not generic descriptions
5. **Customize to agent's domain** and expertise

### Key Placeholders to Replace

- `[Agent Type]`: Full name (e.g., "Product Manager", "Chief Financial Officer")
- `[agent-name]`: Kebab-case name (e.g., "product-manager", "cfo")
- `[primary function]`: Main purpose (e.g., "strategic financial planning")
- `[target audience/industry]`: Who uses this (e.g., "technology companies")
- `[Capability Category X]`: Major capability areas
- `[Use Case Title]`: Specific scenario names

### Section Guidelines

**What This Agent Does**: 5-7 high-level capabilities
**How to Use**: Clear invocation instructions
**Agent Capabilities**: 4-5 groups with 3-4 items each
**Reference Materials**: List actual files created
**Example Use Cases**: 3-5 realistic scenarios with process
**Expertise Areas**: Industries, stages, domains
**Personality & Approach**: Communication and principles
**Best Practices**: 5 tips for effective use
**Technical Foundation**: Credibility builders
**Limitations**: What can/cannot do
**Quick Start Examples**: 8-10 ready prompts

### Quality Standards

**Good Example - Use Case**:
```markdown
### 2. Quarterly Planning Session
**You**: "I need to create our Q2 2024 product roadmap with OKRs"

**Agent will**:
- Analyze current product metrics and user feedback
- Review competitive landscape and market trends
- Generate prioritized feature list using RICE framework
- Create OKRs aligned with company strategy
- Develop timeline with dependencies and milestones
- Prepare stakeholder communication plan

**Result**: Complete Q2 roadmap with 3-5 OKRs, prioritized backlog, and presentation materials
```

**Good Example - Capability**:
```markdown
### Strategic Planning & Roadmapping
- Quarterly and annual product roadmap development
- OKR setting and cascade alignment
- Feature prioritization using RICE/WSJF frameworks
- Dependency mapping and risk assessment
- Stakeholder communication and buy-in strategies
```

**Good Example - Quick Start**:
```markdown
"Help me prioritize these 15 feature requests using the RICE framework"

"Create a product roadmap for our SaaS platform for the next 6 months"

"I need to write a PRD for our new analytics dashboard feature"
```

### Common Mistakes to Avoid

❌ **Too Generic**: "Helps with product management tasks"
✅ **Specific**: "Creates quarterly roadmaps with prioritized features using RICE scoring"

❌ **Vague Use Case**: "You ask for help, agent provides assistance"
✅ **Detailed Use Case**: Shows exact steps and deliverables

❌ **No Examples**: "Request templates as needed"
✅ **With Examples**: Lists specific templates available

❌ **Generic Limitations**: "Cannot replace a human"
✅ **Specific Limitations**: "Cannot access real-time user analytics data"

### Length Requirements

To achieve 250+ lines:
- Expand use cases with detailed steps
- Include more capability details
- Add comprehensive quick starts (10+ examples)
- Elaborate on best practices
- Include detailed expertise areas

### Validation Checklist

Before finalizing:
- ☐ All placeholders replaced
- ☐ 250+ lines minimum
- ☐ 3-5 detailed use cases
- ☐ 8-10 quick start prompts
- ☐ Reference materials match actual files
- ☐ No generic descriptions
- ☐ Professional tone
- ☐ Clear invocation instructions
- ☐ Specific limitations stated
